## &#127775; Featured Work

### &#127968; RentHub
Modern rental platform for browsing, comparing, and booking properties with a product-oriented user experience.

<p>
  <a href="https://github.com/janmej0y/RentHub"><img src="https://img.shields.io/badge/Repo-RentHub-111827?style=for-the-badge&logo=github" alt="RentHub Repo" /></a>
  <a href="https://rent-hub-two.vercel.app/"><img src="https://img.shields.io/badge/Live%20Demo-16a34a?style=for-the-badge&logo=vercel&logoColor=white" alt="RentHub Live Demo" /></a>
</p>

<p align="center">
  <img src="./assets/renthub.gif" alt="RentHub Preview" width="100%" />
</p>

**Problem**  
Rental search products often feel fragmented, cluttered, or incomplete across discovery, comparison, and booking.

**Build**  
I built a polished rental platform with discovery filters, compare flow, booking workflow, payment proof upload, profile handling, and receipt generation.

**Why it stands out**

- &#10024; Advanced discovery with filtering, compare flow, saved items, and recently viewed history
- &#129534; Booking workflow with payment proof upload, agreement consent, and printable receipts
- &#128737;&#65039; Local fallback logic for critical flows when the backend or network becomes unreliable
- &#129513; Reusable components, hooks, and typed models organized for maintainability

**Built with:** `Next.js 15` `TypeScript` `React 19` `Tailwind CSS` `shadcn/ui` `Supabase`

### &#128499;&#65039; EzeeVote Online Voting System
Secure digital election platform centered on verification, vote integrity, and practical admin control.

<p>
  <a href="https://github.com/janmej0y/Online-Voting-System"><img src="https://img.shields.io/badge/Repo-EzeeVote-111827?style=for-the-badge&logo=github" alt="EzeeVote Repo" /></a>
  <a href="https://online-voting-system-henna.vercel.app/"><img src="https://img.shields.io/badge/Live%20Demo-16a34a?style=for-the-badge&logo=vercel&logoColor=white" alt="EzeeVote Live Demo" /></a>
</p>

<p align="center">
  <img src="./assets/ezeevote.gif" alt="EzeeVote Preview" width="100%" />
</p>

**Problem**  
Voting systems need trust, clear verification, and manageable admin workflows, not just basic CRUD.

**Build**  
I built an election platform with identity-oriented verification, admin tooling, lifecycle management, and vote integrity rules.

**Why it stands out**

- &#128274; Google Sign-In, JWT sessions, and protected admin routes
- &#128248; Verification flows with identity capture and trust-oriented account setup
- &#128450;&#65039; Election lifecycle management, candidate CRUD, CSV import, logs, and exports
- &#9989; One-vote-per-election enforcement with receipts and results tooling

**Built with:** `HTML` `CSS` `JavaScript` `Node.js` `Express` `SQLite` `JWT` `Multer` `Chart.js`
